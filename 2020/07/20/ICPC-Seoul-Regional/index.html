B Gene Tree

题意

对一棵树的 $n\leq 100,000$ 叶子节点，两两求距离的平方。

解法一:树形DP

考虑传递平方和。

将当前的点的值传递到下一个点，经过边长为 $w$ 。那么我们需要维护的值最终的值是平方和，记为

$\sum\limits_{p\in Leaf} d_{now}[p]^2$，其中 $d[p]$ 是 $p$ 到当前节点的距离， $Leaf$ 是已经经过的叶子节点的集合。那么我们传递到下一个点时，维护的值就应该变为

$\sum\limits_{p\in Leaf} d_{son}[p]^2 =\sum\limits_{p\in Leaf} (d_{now}[p] + w)^2 = \sum\limits_{p\in Leaf}d_{now}[p]^2 + 2\cdot w\cdot \sum\limits_{p\in Leaf}d_{now}[p] + w^2\cdot|Leaf|$

也就是说，我们需要记录的值还有 $\sum\limits_{p\in Leaf} d_{now}[p]$ 和 $|Leaf|(Leaf的元素个数)$ ，DP时处理一下即可。

注意在处理完叶子节点后要更新父节点的相关值。

时间复杂度O(n)

D - Ladder Game

题意

有 $n-1$ 个梯子拼在一起, 每个梯子有最多 $L$ 个阶梯. 不会有相邻的两个梯子在同一水平线上都有阶梯. 想象有 $n$ 个珠子编号 $1-n$ 沿着 $n$ 条竖线下落, 遇到一个阶梯时就沿着阶梯移动到另一条竖线(类似伽尔顿板). 最后得到的珠子从左往右是一个排列 $\pi$. 现在要移除尽可能多的阶梯, 使得最后得到的排列不变, 输出最后保留的阶梯. $n\leq 50, L\leq 1000$.

题解

按高度排序, 每个阶梯的作用相当于交换两个相邻的珠子(称作置换$(a_i,a_{i+1})$). 给珠子编号后, 我们模拟这个过程, 如果发现当前交换的两个珠子形成逆序, 即 $a_i > a_{i+1}$, 就找到前面是哪一步发生了置换 $(a_{i+1},a_i)$, 把这一步和当前步删掉, 重复此过程, 就完了. 下面说明正确性.

首先置换 $(a_{i+1},a_i)$ 是存在的, 因为我们总是交换两个相邻的数字, 要存在 $a_i > a_{i+1}$, 必然有置换 $(a_{i+1},a_i)$. 那么, 为什么这样不会改变最后的排列呢, 看图~.

然后考虑这样操作后为什么保留的阶梯数是最少的. 经过这一系列操作, 就不存在置换 $(a_i,a_{i+1}),a_i>a_{i+1}$, 即每次置换都会使得全局逆序数加一; 同时, 不存在更少的操作数来达到这个逆序数. 所以此时保留的阶梯是最少的.

E - Network Vulnerability

题意

给出 $n$ 个区间, $n\leq 2000$, 每个区间代表一个顶点, 如果两个区间相交, 它们代表的顶点之间就连一条无向边. 定义 $c(i)$ 为移除 $i$ 个点后最大的联通块数. 试输出 $c(0), c(1), \cdots, c(n-1)$.

题解

首先有个无关的事情. 就是由区间生成的图并没有什么特殊性质, 主要是可以给你 $O(n)$ 的数据来表示 $O(n^2)$ 条边. $CF$ 出过由一个树推导出这个区间的题.

我这题基本是看代码揣测做题人的意图, 所以后面附上他的加了注释的代码.

离散化, 对区间按左端点排序. 定义 $dp[i][j][k]$ 为利用前 $i$ 个区间, 形成 $j$ 个联通块, 整体右端点为 $k$ 时, 最多能剩几个区间.

转移的时候就是枚举前面的区间右端点与当前区间的位置关系, 这个比较自然,, 可以直接看代码.

G Same Color

题意

x轴上有 $n \leq 10^5$ 个点，坐标在 $[0, 10^9]$ 之间，每个点有一种颜色。

现在把点分为 $A,B$ 两组，要求对于B组里的点而言，离它最近的一个A组的点必须和它是一个颜色的（如果有两个点都离它最近，那么其中一个和它颜色一致即可），求A组的点的个数最少是多少。

题解:动态规划+线段树

显然：每一段点的颜色相同的区间上必有至少一个点被分到A组

首先，将整个 $x$ 轴分为若干个颜色相同的区间

$L[c]$ 表示第c个颜色区间的最左侧的点的标号

$R[c]$ 表示第c个颜色区间的最右侧的点的标号

$dp[i]$ 表示第 $i$ 个点之前至少有多少个点被分配到了A组

那么对于一个点 $i$ ，我们考虑两种状态

1、它不是当前同一颜色区间的最后一个点，那么它后面的所有的在当前这个颜色区间的点 $j$ 都可以作为当前区间的最后一个点，

$dp[j] = min(dp[j], dp[i] + 1)，j\in i,R[c_i]$

2、它是当前同一颜色区间的最后一个点，那么在 $[x_i + 1, 2\cdot x_{R[c_i]} - x_i)$ 中都不能出现下一个颜色的A点，而在 $[2\cdot x_{R[c_i]} - x_i,2\cdot x_{L_{c_i + 1}}-x_i]$ 中必然出现下一个颜色的A点

因此在区间合法的情况下（这个区间必须被包含在下一个颜色区间内）

$dp[j] = min(dp[j],dp[i]+1),j\in [2\cdot x_{R[c_i]}-x_i,2\cdot x_{L_{c_i + 1}} - x_i]$

用线段树维护区间最小值

H Strike Zone

题意

平面内给定 $n_1\leq 1000$ 个权值为 $c_1$ （正权值）的点， $n_2 \leq 1000$ 个权值为 $-c2$ （负权值）的点。找一个矩形区域，是区域内点权值之和最大，输出最大权值。

坐标 $(x, y)$ 满足 $-10^9 \leq x,y \leq 10^9$

题解:离散化+线段树

首先，离散化所有点，得到一个最大 $2000*2000$ 的矩阵区域。

DP求最大子矩阵和 $O(n^3)$。

但是实际上，绝大多数的点的值为0，对结果无影响，真正能够对结果产生影响的点只有 2000 个。

将DP的思想提炼一下：

将i到j行的值进行叠加，将二维矩阵一维化，然后维护一个区间最大值。

枚举第i行到第j行的过程是无法简化的，但是这个过程实际上是 $n^2$ 的，可以接受。

考虑用线段树维护区间最大值，枚举统计的起点 $i$，依次加入 $i$ 后每一行的点（纵坐标相同的点），每一行加入后统计答案（相当于对于起点第i行枚举到第j行），点的离散分布决定了这个过程的次数是与枚举 $j$ 同阶的（插入O(n) 个点），线段树插入单个点复杂度为 $O(logn)$ ，最终时间复杂度为 $O(n^2logn)$

线段树维护

线段树上第 $x$ 个点存着第 $i$ 行到第 $j$ 行的第 $x$ 列的权值和。

最终需要维护的值为 $maxsum$ （区间最大值）

为了维护 $maxsum$ 我们需要维护 $lsum(区间左侧最大值), rsum(区间右侧最大值),totsum(区间总和)$

具体操作见代码。

I Thread Knots

题意

在 $x$ 轴上存在 $n\leq 10^5$ 条线段，每条线段上有一个点，可以在线段上的整数点左右移动。

线段满足：不存在一条线段被另一条完全覆盖 $(s_a\leq s_b \&\& e_b\leq e_a)$。

求一种点分布方式，使得相邻点之间的最小距离最大。

题解:二分

二分一个答案，验证时尽量将点向左放

K Washer

题意

给定三维平面里的 $n\in[1,100]$ 个点 $\{(r_i, g_i, b_i)\},r,g,b\in[0,1000]$ ，每次处理一些点时会产生 $r = \frac{\sum_{i = 1}^n r_i}{n},g = \frac{\sum_{i = 1}^n g_i}{n},b=\frac{\sum_{i = 1}^n b_i}{n}$ ，最终产生一个代价 $c = \sum\limits_{i = 1}^n(r_i - r)^2 + (g_i - g)^2 + (b_i - b)^2$ ，现在给定 $k\in[1,2]$ 次处理机会，将所有点处理完，求最小代价。

题解:计算几何

抄了一波AK题解。

每次处理的代价实际上就是点集的方差，其实就是将点集划分，使得方差最小。

k=1时无需讨论。

k=2时，显然点与重心离得越近越好，因此如果将划分后的每个点集处理出凸包，那么两个凸包必定是不相交的。

同时，必然存在一个平面，与两个凸包分别相切（此处的相切是指最多有一根直线相交），且将两个凸包分隔在两侧。

考虑枚举三个点构成这个平面，点集划分后，再将三个点分别分配到两个点集中（因为在相切的平面上，所以可能属于任何一边），然后处理出最小的结果，复杂度 $O(n^4)$。

L What’s Mine is Mine

题意

给定 $m\leq 100$ 种矿的价格 $pri[i] \in[1,10,000]$， $n\leq 10,000$ 个矿藏，包含矿藏的挖掘时间 $s,e,0<s<e<15,000$ ，矿藏种类 $k$ 。矿的价值为 $(e - s)\cdot pri[k]$ 。每次挖掘矿藏时必须从 $s$ 挖到 $e$ ，中途不能停止，或者中途开始挖掘矿藏。问能够挖掘到的最大价值时多少。

题解:排序+DP

按 $e$ 排序。

$dp[i]$ 存 $i$ 时刻能够赚到的最多的钱， $dp[e] = max(dp[e], dp[s] + (e - s) * pri[k]) $
