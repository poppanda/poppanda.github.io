H Strike Zone

题意

平面内给定 $n_1\leq 1000$ 个权值为 $c_1$ （正权值）的点， $n_2 \leq 1000$ 个权值为 $-c2$ （负权值）的点。找一个矩形区域，是区域内点权值之和最大，输出最大权值。

坐标 $(x, y)$ 满足 $-10^9 \leq x,y \leq 10^9$

题解:离散化+线段树

首先，离散化所有点，得到一个最大 $2000*2000$ 的矩阵区域。

DP求最大子矩阵和 $O(n^3)$。

但是实际上，绝大多数的点的值为0，对结果无影响，真正能够对结果产生影响的点只有 2000 个。

将DP的思想提炼一下：

将i到j行的值进行叠加，将二维矩阵一维化，然后维护一个区间最大值。

枚举第i行到第j行的过程是无法简化的，但是这个过程实际上是 $n^2$ 的，可以接受。

考虑用线段树维护区间最大值，枚举统计的起点 $i$，依次加入 $i$ 后每一行的点（纵坐标相同的点），每一行加入后统计答案（相当于对于起点第i行枚举到第j行），点的离散分布决定了这个过程的次数是与枚举 $j$ 同阶的（插入O(n) 个点），线段树插入单个点复杂度为 $O(logn)$ ，最终时间复杂度为 $O(n^2logn)$

线段树维护

线段树上第 $x$ 个点存着第 $i$ 行到第 $j$ 行的第 $x$ 列的权值和。

最终需要维护的值为 $maxsum$ （区间最大值）

为了维护 $maxsum$ 我们需要维护 $lsum(区间左侧最大值), rsum(区间右侧最大值),totsum(区间总和)$

G Same Color

题意

x轴上有 $n \leq 10^5$ 个点，坐标在 $[0, 10^9]$ 之间，每个点有一种颜色。

现在把点分为 $A,B$ 两组，要求对于B组里的点而言，离它最近的一个A组的点必须和它是一个颜色的（如果有两个点都离它最近，那么其中一个和它颜色一致即可），求A组的点的个数最少是多少。

题解:动态规划+线段树

显然：每一段点的颜色相同的区间上必有至少一个点被分到A组

首先，将整个 $x$ 轴分为若干个颜色相同的区间

$L[c]$ 表示第c个颜色区间的最左侧的点的标号

$R[c]$ 表示第c个颜色区间的最右侧的点的标号

$dp[i]$ 表示第 $i$ 个点之前至少有多少个点被分配到了A组

那么对于一个点 $i$ ，我们考虑两种状态

1、它不是当前同一颜色区间的最后一个点，那么它后面的所有的在当前这个颜色区间的点 $j$ 都可以作为当前区间的最后一个点，

$dp[j] = min(dp[j], dp[i] + 1)，j\in i,R[c_i]$

2、它是当前同一颜色区间的最后一个点，那么在 $[x_i + 1, 2\cdot x_{R[c_i]} - x_i)$ 中都不能出现下一个颜色的A点，而在 $[2\cdot x_{R[c_i]} - x_i,2\cdot x_{L_{c_i + 1}}-x_i]$ 中必然出现下一个颜色的A点

因此在区间合法的情况下（这个区间必须被包含在下一个颜色区间内）

$dp[j] = min(dp[j],dp[i]+1),j\in [2\cdot x_{R[c_i]}-x_i,2\cdot x_{L_{c_i + 1}} - x_i]$

用线段树维护区间最小值

B Gene Tree

题意

对一棵树的 $n\leq 100,000$ 叶子节点，两两求距离的平方。

解法一:树形DP

考虑传递平方和。

将当前的点的值传递到下一个点，经过边长为 $w$ 。那么我们需要维护的值最终的值是平方和，记为

$\sum\limits_{p\in Leaf} d_{now}[p]^2$，其中 $d[p]$ 是 $p$ 到当前节点的距离， $Leaf$ 是已经经过的叶子节点的集合。那么我们传递到下一个点时，维护的值就应该变为

$\sum\limits_{p\in Leaf} d_{son}[p]^2 =\sum\limits_{p\in Leaf} (d_{now}[p] + w)^2 = \sum\limits_{p\in Leaf}d_{now}[p]^2 + 2\cdot w\cdot \sum\limits_{p\in Leaf}d_{now}[p] + w^2\cdot|Leaf|$

也就是说，我们需要记录的值还有 $\sum\limits_{p\in Leaf} d_{now}[p]$ 和 $|Leaf|(Leaf的元素个数)$ ，DP时处理一下即可。

注意在处理完叶子节点后要更新父节点的相关值。

时间复杂度O(n)

解法二:点分治

这道题求的是点和点之间的距离，显然满足点分治的要求（虽然比赛的时候没想过要用这个东西）。

对于每一层分治，我们只需要储存一个整体的距离平方和、距离和、叶子节点数即可。

时间复杂度 $ O(nlogn)$

拓扑排序

引入

对于一种执行的依赖关系：

要执行B，先执行A

我们可以使用一条边 $A\to B$ 表示。即想要执行B，必须先把所有指向A的点执行了，当 B的入度为0时 ，B才可以被执行。

例如，现在有边 $A\to B, B\to C, D \to B$ ，表示想要执行B，先要执行 A,D​ ， 想要执行 C ，先要执行B

那么执行的顺序为 $A,D,B,C$ ，或者 $D,A,B,C$

对这种执行先后顺序的处理，我们称为拓扑排序。

处理方法

容易看出，只有当一个点的入度为0时，它才没有依赖的点，那么它就是可执行的。

处理过程

用stack储存入度为0的点

循环：

​ 1、从stack中取出一个点，并将它输出到排序结果中

​ 2、将该点指向的所有点的入度减一（依赖的点减少一个）

​ 3、将入度减一后入度为0的点加入到stack中

当stack为空，表示入度为0的点没有了，没有可以执行的点，算法结束

如此一来，如果图中没有剩下的点，那么我们就可以得到一个合法的拓扑排序。如果图中还有点，说明图内存在环（如下图），无法生成拓扑排序。

字典序最小的拓扑排序

可以使用小根堆（或者叫做优先队列 priority_queue）代替stack。它每次会将储存的元素中最小的元素取出，因此我们能保证每次从入度为0的点集中取出的元素字典序最小

欧拉回路和欧拉通路

欧拉回路的定义：对于一张图，能够从起点出发，把每条边走且仅走一遍，回到起点的路径称为欧拉回路

欧拉通路的定义：对于一张图，能够从起点出发，把每条边走且仅走一遍的路径称为欧拉通路

欧拉回路的性质：

​ 有向图：图连通，所有的顶点出度=入度。

​ 无向图：图连通，所有顶点都是偶数度。

欧拉通路的性质：

​ 有向图：图连通，有一个顶点出度大入度1，有一个顶点入度大出度1，其余都是出度=入度。

​ 无向图：图连通，只有两个顶点是奇数度，其余都是偶数度的。

F.Lambda Calculus

题意

定义一种表达式 lambda(x) ，其中x变量称为自变量，在与lambda同一级括号或更低级的表达式中出现的 x 均受该表达式的中自变量的限制，求出对于一串表达式，其中的非自变量有哪些（字典序输出）

输入：表达式长度 $n \leq 10^7$ ，表达式中的变量和常量均由字母和连字符-（长度可能不止为1）组成，且字符串中会有多余的不对结果产生干扰的空格

例如：

lambda( x)(x y) , (x y) 与lambda(x) 在同一级，故括号内的 (x y) 中 x 为自变量， y为非自变量

lambda(y)(lambda(z)(x y z))中，(lambda(z)(x y z))与 lambda(y)在同一级，而(x y z) 则在更低一级，与lambda(z)同级，故(x y z)中，x为非自变量，y，z均为自变量

题解

类似深搜的思想

只需要按照括号分级即可，对于lamda(x)的括号特殊处理，将其中的值进行记录，而对于其它括号，进入下一级进行处理，处理完当前这一级和更低级之后，将记录的当前级的自变量删除即可

I.Travel Brochure

题意

给定一串长度为 $n$ 数列 $w_0,w_1,w_2,…,w_{n-1}$ ，若 $sum_i = \sum\limits_{i=0}^i w_i$ ，定义 $a_i = w_i, b_i = sum_i$

求若 $m\geq 3$ 个点 $p1 ,p2,…,p_m$ 满足 $p_i < p_{i + 1}$ ，使得 $\frac{1}{2}\sum\limits_{i=0}^m (a_{p_{i + 1}}-a_{p_i})\frac{b_{p_{i+1}}b_{p_i}}{a_{p_{i+1}}a_{p_i}}$ 最大，输出最大值

题解

将 $(a_{p_{i + 1}}-a_{p_i})\frac{b_{p_{i+1}}b_{p_i}}{a_{p_{i+1}}a_{p_i}}$ 化为 $\frac{b_{p_{i+1}}b_{p_i}}{a_{p_i}} - \frac{b_{p_{i+1}}b_{p_i}}{a_{p_{i+1}}}$ = $(\frac{b_{p_i}}{a_{p_i}}, b_{p_i}) \times (\frac{b_{p_{i + 1}}}{a_{p_{i+1}}}, b_{p_{i+1}})$ （向量叉积）

乘上 $\frac{1}{2}$ ，即为三角形 $(0,0),(\frac{b_{p_i}}{a_{p_i}}, b_{p_i}),(\frac{b_{p_{i + 1}}}{a_{p_{i+1}}}, b_{p_{i+1}})$的面积

题目也就变成了求点集 $\{(\frac{b_{p_i}}{a_{p_i}}, b_{p_i})\}$ 中最大凸包的面积

J.Cliques

题意

给定一个包含 $n(1 \leq n \leq 100)$ 个点的无向图，每一步操作可以在两点间增加/删除一条边，问能否在10步之内将这张图变为若干个完全子图的集合(即每一个顶点都在一个完全图中)

题解

考虑完全图的性质

对于任一完全图，其顶点 $V$ 集合的任一子集 $v\subseteq V$ 所在子图也为完全图 (显然)

对于顶点数 $n\leq 2$ 的图，一定为完全图

对于顶点数 $n \geq 3$ 的图，每次讨论三个顶点。

1、若当前三个顶点在同一完全图中，则三个顶点两两之间一定相连，这种情况不予讨论

2、若当前三个顶点间只存在两条边，则说明通过增边/删边操作可将当前三点构成的子图变为完全图/完全图集合

3、若当前三个顶点间只存在1/0条边，则三点构成的子图已经为完全图集合([a,b] + [c] / [a] + [b] + [c])，不予讨论

对于情况2，我们增加/删除一条边，即可将其转化为情况1/3

也就是说，我们要将所有三点子图转化为完全图，当所有三点子图均为完全图/完全图集合后，其构成的图一定为完全图/完全图集合

解法：深搜

每次搜索不完全三点子图，对增减边情况(三条边依次)分别讨论

复杂度

搜索最坏时间复杂度 $O(3^{10})$

在搜索过程中处理三元环最坏时间复杂度为 $O(n^3)$ ，总时间复杂度最坏为 $O(n^3\cdot 3^{10})$ ，但是在随机化数据中可以轻松通过

预处理三元环时间复杂度为 $O(n^3)$ ，在搜索过程中修改边后修改三元环复杂度为 $O(n)$ ，三点非完全子图数量最大为10，时间复杂度为 $O(n^3 + 3^{10}\cdot(10 + n))$ ，时间复杂度可接受，代码复杂度不可接受

K.Finding Hotels

题意

二维平面内存在 $N(N\leq 2\cdot 10^5)$ 个点，每个点有一个权值 $C$

$m(m\leq 2\cdot 10^4)$ 次询问，每次给定 $x, y, c$ ，求平面内离 $(x,y)$ 最近的权值 $C \in [1,c]$ 的点，距离权值相等时取先出现的点

题解

解法一: 在线+三维KD-Tree

将 $C$ 作为一维，在查询时遇到 $C$ 的最小值大于 $c$ 的区间就直接剪枝，而查询 $x,y$ 时则按照 KD-Tree 的正常查询思路查找即可

该思路十分卡常，需要优秀的KDTree板子

解法二: 离线+动态KD-Tree

按照权值从小到大向KD-Tree中加入节点，加入节点的顺序由按询问中 $c$ 的升序排列决定

使用替罪羊树的思想，对失衡子树重构，保证效率

该思路显然会更快，但是需要更优秀的可插入节点的KDTree的板子

Master of Subgraph

题意

给定一棵树($n\in [1,3000]$)，每个点有权值($v\in[1, 10^5]$)。给定一个值$m\in[0, 10^5]$，问对于所有$i\in[1,m]$，是否存在连通子图，其权值之和等于$i$

解法：点分治+bitset

bitset第 $i$ 位存储对于当前点，是否存在包含它的权值之和为 $i$ 的连通子图

点分治，对于每一次分治过程，当前过程的子树的根必须被包含在方案中，因此从根 $rt$ 向下遍历时，子节点 $i$ 的初始值为 $v[rt] << w[i]$，表示选择了子节点及包含根的子图后的可行方案。子节点DP完成后，$rt$ 执行 $v[rt] |= v[i]$，将子节点的方案包含进来

不包含当前子树的根的方案的计算过程在下一层分治中（下一层分治必定不包含当前分治的根）

最后的结果即为整棵树的根记录的结果

Marriage

题意

有 $n\in[1,10^5]$ 个家庭，对于家庭 $i$ ，有儿子 $a_i$ 个， 女儿 $b_i$ 个，满足 $sum = \sum\limits_i a_i = \sum\limits_i b_i \in[1,10^5]$

现在给这些子女配对（一男配一女），同一家庭的孩子不能配对

问全部配对完有多少种方案($\mod 998244353$)

解法：容斥+NTT

考虑容斥，$V_i$ 表示至少有 $i$ 对不合法配对的方案数，则结果为 $\sum\limits_{i=0}^{sum}(-1)^iV_i$

对于单独一个家庭，$V_i=C_a^i\cdot C_b^i\cdot i!(i\in[0,min(a,b)])$

对于两个家庭$V(1)、V(2)$，将其合并为$V_i = \sum[x+y==i]V(1)_{x}\cdot V(2)_{y}$

对于多个家庭反复两两合并即可

考虑NTT，对于第 $i$ 个家庭，其方案数用多项式

$V(i) = V(i)_0\cdot x^0 + V(i)_1\cdot x^1…$

$x^i$ 的系数表示不合法配对的个数至少为 $i$ 的方案数

多项式相乘，记得到两个家庭合并的结果
